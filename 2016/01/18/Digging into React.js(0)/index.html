Since, recently, a few guys from US started telling me that I should really make my Blog English-based. Since most of the readers are actually pretty well English reader and shit tons of English Docs about everything but rarely too much Chinese Docs. Well, English then. I am actually a terrible English speaker. So, I would try to make these posts at least understandable for both Chinese and English speakers.

Well, brief into about React.js. It’s a really fancy front-end framework published by Facebook(TM) @ 2015. Well, in fact, it’s not actually well-qualified to be called a framework. Unlike these mature MVC front-end framework. React.js is only the View Layer. The most striking feature of it for me is its really unique measurement of updating the DOM. As we all know, updating DOM is like a really slow process for the browser, especially if the webpage is like a huge huge form for user to submit. Changing a single node in the DOM Tree is almost do the rendering again completely, which is slow and creates really terrible user experience. So, engineers from Facebook use something called Virtual DOM to tackle this problem. The Virtual DOM is stored in the memory and every change of the React comportment would only change the DOM in the memory. Later, a diff system would compare the Virtual DOM to the really one and would only re-render the parts changed.

I am like super new to this new Tech. Due to the excessively lake of Chinese Doc, it’s really super hard to learn. I only know the basic usage. It may even take me months to master even a few mods of it. So, I am trying to keep record of what I learn for other people and for my future review.

I would use JSX and ES6 syntax to write code. Webpack to pack resource up. Try to use Mocha to running test(never use it before lolololol). Node-inspector to debug(never use it before lololol). Using React Router as the router system. Use Nodejs Express as the testing server-side. MongoDB or even try Redis as the database. Possibly using Material-UI or Bootstrap in React.js.

The whole learning process would be me trying to build a whole system of a API server for the service of all of my current webpage and a front-end-only static single page Blog system. First, I hope I could basically wrap up the building of the blog system in two weeks. So, lets begin work tonight!
